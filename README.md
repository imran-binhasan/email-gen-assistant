# Email Generation Assistant

AI-powered professional email generator with a custom evaluation framework comparing two prompting strategies on the same model.

---

## Project Overview

This project fulfills the AI Engineer Candidate Assessment by building a working email generation prototype and a rigorous evaluation system.

- **Generator**: Takes `Intent`, `Key Facts`, and `Tone` → outputs a polished professional email
- **Prompting**: Role-Playing + Few-Shot Examples + Dynamic Tone Hints (three distinct techniques)
- **Evaluation**: 3 custom metrics, LLM-as-a-Judge scoring, 10 scenarios, comparative analysis
- **Comparison**: Advanced prompt strategy vs Zero-shot baseline, same model (Gemini 3.1 Flash Lite)

## Prompt Engineering Techniques

Three distinct techniques are layered together in the Advanced strategy:

### 1. Role-Playing (System Prompt)
The model is given a persona: *"expert professional email writer with 15 years of experience."* This anchors tone, quality, and behavioral rules from the first token. Hard rules (`Never add facts not provided`, `Always start with Subject:`) prevent the most common failure modes.

### 2. Few-Shot Examples
Two complete, high-quality example emails are injected before every task — one formal/confident, one urgent/direct. These span opposite tone extremes so the model always has a nearby reference regardless of what tone is requested. Empirically, these eliminate the tendency to default to corporate boilerplate.

### 3. Dynamic Tone Hints (Chain-of-Thought style nudge)
A `_tone_hint()` function detects tone keywords and injects targeted per-call guidance for tones most prone to model drift:
- **Casual**: *"Never write 'I am writing to' or 'I came across your company'"*
- **Urgent/Direct**: *"No warm opener. No 'Best regards'. End with a brief status line."*
- **Apologetic**: *"Open with 'I sincerely apologize'. Take full responsibility before explaining."*

This is the key differentiator. Without it, models default to formal/corporate language regardless of tone request.

### Baseline (Comparison Strategy)
A zero-shot approach using only a minimal system prompt and no examples or tone hints. Same model, same temperature — isolates the contribution of the advanced techniques.

## Custom Evaluation Metrics

### Metric 1 — Fact Recall Score

**Definition:** Fraction of input facts that are present (verbatim or paraphrased) in the generated email.

**Logic:**
- LLM-as-a-Judge (Gemini 3.1 Flash Lite, temperature=0) checks each fact individually
- Semantic matching: facts count as present even when paraphrased
- Score = `facts_present / total_facts` (0.0 → 1.0)
- Returns structured JSON: per-fact verdict + overall score + justification

**Why this matters:** A beautiful email that omits a key deadline or discount offer is useless. This metric directly measures the model's ability to incorporate all provided information.

---

### Metric 2 — Tone Alignment Score

**Definition:** How well the generated email sustains the requested tone throughout.

**Logic:**
- LLM-as-a-Judge with a structured scoring rubric (1.0 / 0.75 / 0.5 / 0.25 / 0.0)
- Judge must provide a one-sentence justification — prevents arbitrary scoring
- Scores: 1.0 = perfect throughout; 0.75 = mostly correct with minor slip; 0.5 = partially correct; 0.25 = mostly wrong; 0.0 = completely wrong

**Why this matters:** Tone mismatch is the most visible quality failure. A "casual" email that reads like a legal document, or an "urgent" email that opens with "I hope this message finds you well," undermines the entire purpose.

---

### Metric 3 — Fluency Score (Hybrid)

**Definition:** Combined professionalism/fluency rating from LLM judgment and structural similarity to the human reference.

**Logic:**
- **Component A (LLM):** Judge rates grammar, professionalism, and writing quality (0.0–1.0)
- **Component B (ROUGE-L):** Automated longest-common-subsequence overlap against the human reference email
- **Final score:** `(llm_score + rouge_l_score) / 2`

**Why this matters:** The LLM judge alone can be lenient; ROUGE-L anchors the score to a human-written reference. The hybrid approach is methodologically richer — one automated metric and one learned judgment, averaged together.

## Comparative Analysis

### Which strategy performed better?

**Advanced prompt (0.8925 overall) outperforms Baseline (0.853)** — a +4.6% improvement in overall score, with perfect scores on both Fact Recall and Tone Alignment across all 10 scenarios.

### Biggest failure mode of the Baseline

**Tone calibration on non-default tones.** The zero-shot baseline has no reference for what "casual" or "urgent" looks like in practice, so it defaults to formal/corporate language regardless of the tone requested. This is most visible in:

- **S3 (Casual, persuasive):** Baseline scored 0.629 (tone=0.5, fact recall=0.75). The model wrote *"I came across your company and thought our AI-powered HR automation tool could be a great fit"* — stiff, corporate phrasing. It also missed one of the four facts. Advanced scored 0.896 (tone=1.0, fact recall=1.0), generating *"Hey, hope your week's going well. We built an AI-powered HR tool that's currently helping teams cut onboarding time by 40%."*

- **S7 (Enthusiastic, professional):** Baseline tone=0.75 — enthusiasm only "mildly conveyed." Advanced tone=1.0.

- **S8 (Urgent, direct):** Baseline tone=0.75 — added warm opener and "Best regards" close. Advanced tone=1.0 — led directly with the problem, ended with "Will keep you posted."

### Production Recommendation

**Recommend the Advanced prompt strategy** for the following reasons:

1. **Perfect fact recall (1.0 vs 0.975):** Every provided fact appears in every email. Missed facts in production emails are a business error.

2. **Perfect tone alignment (1.0 vs 0.9):** The 10% gap is not uniform — baseline fails hard on non-standard tones. Casual and urgent tones are common in real-world use (sales outreach, incident response). A production system must handle them.

3. **Comparable fluency:** Both strategies score ~0.68 on the hybrid fluency metric. The Advanced prompt's lower ROUGE-L is expected — casual emails look structurally different from formal reference emails, which is correct behavior, not a deficiency.

4. **Reliability:** Advanced prompt's consistent performance (no scenario below 0.87) vs Baseline's high variance (0.629–0.919 range) makes it more predictable in production.

## Technical Notes

- **Judge model:** Gemini 3.1 Flash Lite (temperature=0) used as single consistent judge for all scenarios. A single judge improves score comparability across strategies.
- **Rate limiting:** 5-second sleep between judge calls, 20-second sleep between scenarios — keeps peak usage at ~6 RPM, well under the 15 RPM free-tier ceiling.
- **Retry logic:** Tenacity exponential backoff on all API calls (4 attempts, 10–90s wait) for robustness.
- **Model:** `gemini-3.1-flash-lite` — 500 RPD free tier, no billing required.
